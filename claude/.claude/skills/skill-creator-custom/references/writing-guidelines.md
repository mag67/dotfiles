# SKILL.md 執筆ガイドライン

## 基本方針

SKILL.md は「Claude への作業指示書」であり、解説文書ではない。読者は高性能 LLM であることを前提に、最小限の指示で最大限の行動を引き出す。

## 行数の目標

| 区分 | 行数 | 方針 |
|------|------|------|
| シンプルなスキル | 40-80行 | SKILL.md のみで完結 |
| 中程度のスキル | 80-150行 | 主要な手順 + references/ |
| 複雑なスキル | 150-200行 | フェーズ概要のみ。詳細は references/ に委譲 |

200行を超える場合は references/ への分離が不十分。500行は許容しない。

## 簡潔に書くための原則

1. **命令形で書く** — 「〜することが推奨されます」ではなく「〜する」
2. **why は1文で十分** — 背景説明は最大1文。理由が自明なら省略する
3. **参照に委譲する** — 手順の詳細、テンプレート、具体例は references/ に置き、SKILL.md からは「`references/xxx.md` を読み込む」と指示するだけにする
4. **繰り返さない** — 同じ指示を別の言い回しで繰り返さない。「念のためもう一度」は削除する
5. **Anti-pattern を書かない** — 「〜してはいけない」の列挙より「〜する」の明示が効果的。Anti-pattern は本当に頻発する誤りに限定する

## 構造テンプレート

```markdown
---
name: skill-name
description: |
  1文目: 何をするスキルか。
  2文目: どういう時に使うか（トリガー句3-5個）。
---

# タイトル

## 起動時
[準備ステップ: 2-5項目]

## 進め方
[核となる手順: 箇条書きまたは番号付きリスト]

## 成果物
[出力ファイルと配置先]
```

## レビュー時のチェックリスト

- [ ] 200行以内か
- [ ] 各セクションが30行以内か
- [ ] references/ に委譲できる詳細が SKILL.md 本体に残っていないか
- [ ] 同じ指示の繰り返しがないか
- [ ] description にトリガー句が3-5個含まれているか
