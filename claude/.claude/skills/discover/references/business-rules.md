# ビジネスルールカタログテンプレート (`business-rules.md`)

## 記載ルール

- **ID**: `BR-001` から連番。サブフェーズ1の業務フローで仮採番したBR-IDを引き継ぐ
- **分類**: Wiegers 5類型（事実 / 制約 / 行為トリガー / 推論 / 計算）
- **EARS要約**: 全ルールに付与する。レビュー入口として使い、厳密性が必要な場合は詳細で補足する
- **詳細**: 分類に応じた形式記法を使う
  - 制約 / 推論: Decision Table
  - 計算: 数式 + Example Table
  - 事実 / 行為トリガー: EARS要約を基本とし、必要時のみ補足
- **起源**: 任意。法令 / 契約 / 社内規定 / ドメイン慣習 / 計算ロジック
- **関連シナリオ**: Layer 1のインライン参照から導出可能なため、手編集の必須項目にしない

### EARS形式の種類

| キーワード | 用途 | パターン |
|-----------|------|---------|
| Ubiquitous | 常に成立する制約 | The system shall {要件} |
| Event | イベント駆動 | When {トリガー}, the system shall {要件} |
| State | 状態依存 | While {状態}, the system shall {要件} |
| Unwanted | 異常系 | If {望ましくない条件}, then the system shall {要件} |
| Optional | 選択的機能 | Where {機能が有効}, the system shall {要件} |

（分類に応じた形式で記述）

| Y | N | （結果B） |
| N | - | （結果C） |

#### 計算の場合: 数式 + Example Table

| 入力A | 入力B | 係数 | 期待結果 |
|-------|-------|------|---------|
| 100 | 10 | 1.1 | 120 |

- **起源**: （任意: 法令 / 契約 / 社内規定 / ドメイン慣習 / 計算ロジック）

- **備考**: （任意: 補足注記・監査メモ）
