# 要求管理表テンプレート (`requirements-table.md`)

## 記載ルール

- **ID**: `REQ-001` から連番。カテゴリが変わっても連番を継続する
- **要求内容**: 「〜できる」の形で記述。主語（アクター）が自明な場合は省略可
- **優先度**（MoSCoW法）: Must（MVP必須）/ Should（MVPに含む、リソース次第で調整可）/ Could（MVP後）/ Won't（対象外）
- **起源**: 要求の出所（PM、ドメインエキスパート、法規制、競合調査 等）
- **担当**: 確認・判断の責任者（部門・役職レベル）
- **根拠**: なぜこの要求が必要か。優先度の判断理由や業務上の背景を簡潔に記述する
- **関連UC**: ユースケースIDを記載。複数の場合はカンマ区切り

## テンプレート

```markdown
# 要求管理表

> 最終更新: YYYY-MM-DD ｜ サブフェーズ2で作成

## 凡例
- **優先度**: Must（MVP必須）/ Should（MVPに含む、リソース次第で調整可）/ Could（MVP後）/ Won't（対象外）

## サマリー

| 優先度 | 件数 |
|--------|------|
| Must | X件 |
| Should | X件 |
| Could | X件 |
| Won't | X件 |
| **合計** | **X件** |

---

## （カテゴリ名）

| ID | 要求内容 | 優先度 | 起源 | 担当 | 根拠 | 関連UC |
|----|---------|--------|------|------|------|--------|
| REQ-001 | （〜できる） | Must | （出所） | （責任者） | （なぜ必要か） | UC-001 |

（カテゴリごとにセクションを分けてテーブルを作成する）
```
