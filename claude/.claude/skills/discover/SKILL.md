# Discover: 業務発見フェーズ支援スキル

## ワークフロー

2つのサブフェーズで構成される。PMは必要に応じて行き来できる。

- **サブフェーズ1**: 業務理解の構造化 → 業務フロー図 + ドメインモデル図 + 未確認事項リスト
- **サブフェーズ2**: モックアップ起点の要求整理 → ユースケース一覧 + MVPモックアップ(React) + 要求管理表

---

## 起動時

1. **テンプレートの読み込み**: `references/` ディレクトリ内の該当テンプレートファイルを読み込み、成果物のフォーマットを把握する
2. **保存先の確認**: 成果物は `docs/requirements/` に保存する。PMに別の場所を希望するか軽く確認する
3. **既存成果物の読み込み**: `docs/requirements/` にファイルがあれば読み込み、前回の続きから再開する。初回は空の状態から始める
4. **ドメインの確認**: PMにどの業務ドメインについて整理したいか質問する
5. **ドメインの初期調査**: WebSearchで該当業界の標準的な業務フロー・用語・システム構成を調査し、PMに提示して対話の起点にする
6. **サブフェーズの選択**: サブフェーズ1から始めるか、サブフェーズ2から始めるか確認する

---

## サブフェーズ1: 業務理解の構造化

### 進め方

1. **業務の全体像を聞く**: PMに自由に話してもらい、主要な業務フローを把握する
2. **業務フローの深掘り**: 各ステップの入力・担当・完了条件・例外ケース・判断ルールを明らかにする。PMが答えられない質問は「未確認事項」として記録し先に進む
3. **ドメインモデルの抽出**: 業務フローからエンティティ・関連・値オブジェクト・集約・境界づけられたコンテキストを構築する
4. **定期的な整合性チェック**: 3〜5回のやり取りごとに、フロー間の矛盾・暗黙の前提・業界パターンとの乖離を確認し、PMに指摘する
5. **成果物の生成**: 十分な情報が集まったら成果物を生成してPMに確認する

### サブフェーズ1の成果物

`docs/requirements/` に作成・更新する。フォーマットは `references/` 内の該当テンプレートに従うこと。

- **`business-flow.md`**: 業務フロー図（Mermaid flowchart）、アクター一覧、フローの説明テーブル、業務ルール、ステータス遷移
- **`domain-model.md`**: ドメインモデル図（Mermaid classDiagram）、エンティティ詳細、値オブジェクト、境界づけられたコンテキスト
- **`unconfirmed-items.md`**: 未確認事項リスト（ID付き・優先度付き・確認先付き）

### 成果物の更新タイミング
3回以上のやり取りごとに成果物ファイルを更新する。

## サブフェーズ2: モックアップ起点の要求整理

### 前提
サブフェーズ1の成果物（業務フロー、ドメインモデル）が存在すること。なければサブフェーズ1を先に行う。

### 進め方

1. **モックアップの初期生成**: サブフェーズ1の成果物をもとに、`docs/requirements/mockup/` にReact + Vite + TypeScriptプロジェクトを作成する。WebSearchで該当ドメインのUI/UXパターンを調査し、UIライブラリを選定する。業務フローの主要画面をすべて網羅し、画面間ナビゲーションとダミーデータを実装する
2. **画面ごとのレビュー**: PMと画面ごとにレビューし、フィードバックを収集する
3. **ユースケースの抽出**: 各画面の操作からユースケースを抽出し文書化する
4. **要求の整理と優先度付け**: MoSCoW法（Must/Should/Could/Won't）で優先度付けし、MVP対象を決定する
5. **モックアップの更新**: レビュー結果を反映し、MVP対象の要求に絞った画面構成に調整する

### サブフェーズ2の成果物

`docs/requirements/` に作成・更新する。フォーマットは `references/` 内の該当テンプレートに従うこと。

- **`requirements-table.md`**: 要求管理表（MoSCoW優先度、MVP対象フラグ、カテゴリ別）
- **`usecases.md`**: ユースケース一覧（アクター・事前条件・主フロー・代替フロー・事後条件・関連要求）
- **`mockup/`**: MVPモックアップ（React + Vite プロジェクト）

既存の `domain-model.md` と `unconfirmed-items.md` も必要に応じて更新する。
